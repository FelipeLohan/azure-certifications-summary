
## O que é?
A **Azure Virtual Network (VNet)** é uma rede privada isolada dentro do Azure. É o bloco fundamental de rede — tudo que roda no Azure vive dentro de uma VNet ou se conecta a ela.

## Características principais
- Definida por um **espaço de endereços CIDR** (ex: `10.0.0.0/16`)
- **Isolada por padrão** — nenhuma VNet se comunica com outra automaticamente
- Existe dentro de uma **região** específica do Azure
- Pode ser dividida em **sub-redes (subnets)**

## Subnets
- Segmentam a VNet em partes menores com propósitos diferentes
- Exemplo: subnet pública (front-end web) e subnet privada (banco de dados)
- Cada subnet tem seu próprio bloco CIDR (ex: `10.0.1.0/24`)
- O Azure reserva **5 endereços** em cada subnet para uso interno

## Relação das VMs com a VNet
- Toda VM precisa de uma **NIC (Network Interface Card)** conectada a uma subnet
- A NIC recebe um **IP privado** da subnet (automático ou estático)
- IPs privados são usados para comunicação **dentro** da VNet
- IPs públicos são recursos separados, associados à NIC para acesso externo

## Relação de outros recursos com a VNet
| Recurso | Como se conecta |
|---|---|
| App Service | Via **VNet Integration** |
| Azure SQL / Storage | Via **Service Endpoint** ou **Private Endpoint** |
| AKS | Nodes são VMs dentro de uma subnet |
| Azure Firewall | Implantado em uma subnet dedicada |

## Comunicação entre VNets: VNet Peering
- Por padrão, VNets são isoladas entre si
- **VNet Peering** conecta duas VNets (mesma região ou regiões diferentes — Global Peering)
- Tráfego vai pela rede backbone da Microsoft (baixa latência, sem internet)
- **Não transitivo:** se A↔B e B↔C, A não acessa C automaticamente

## Comunicação com redes on-premises
| Serviço | Como funciona |
|---|---|
| **VPN Gateway** | Túnel criptografado pela internet pública |
| **ExpressRoute** | Conexão privada dedicada (sem internet) |

## Pontos-chave para o exame
- VNet é **regional** — não cruza regiões por padrão
- Sub-redes na **mesma VNet** se comunicam automaticamente
- NSG controla o tráfego **dentro e entre** subnets
