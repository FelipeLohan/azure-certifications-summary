
## Endereço IP

Um endereço IP identifica um dispositivo numa rede. Existem dois tipos:

**IPv4** — formato de 32 bits dividido em 4 octetos (ex: `192.168.1.10`)
**IPv6** — formato de 128 bits em hexadecimal (ex: `2001:0db8::1`) — mais endereços disponíveis

### IPs no Azure: Público vs Privado

| | IP Público | IP Privado |
|---|---|---|
| Alcance | Acessível pela internet | Somente dentro da VNet |
| Atribuído a | VM, Load Balancer, Gateway | VM (NIC), recursos internos |
| Exemplo | `20.10.5.200` | `10.0.1.4` |

### IP Dinâmico vs Estático

| | Dinâmico | Estático |
|---|---|---|
| Comportamento | Muda ao reiniciar o recurso | Permanece fixo |
| Uso | Recursos temporários | DNS, firewall, certificados |

## Portas e Protocolos

- **Porta:** canal lógico de comunicação (ex: porta 80 = HTTP, 443 = HTTPS, 22 = SSH, 3389 = RDP)
- **TCP:** protocolo confiável, com confirmação de entrega (usado em HTTP, RDP, SSH)
- **UDP:** protocolo rápido, sem confirmação (usado em streaming, DNS, jogos)

## CIDR — Notação de Bloco de IP

Notação usada para definir intervalos de endereços de rede:
- `10.0.0.0/16` → 65.536 endereços disponíveis
- `10.0.1.0/24` → 256 endereços disponíveis
- Quanto maior o número após a barra, menor o bloco

## DNS (Domain Name System)

- Traduz nomes (`www.meusite.com`) em endereços IP (`20.10.5.200`)
- Azure oferece o **Azure DNS** para hospedar zonas DNS
- Também há o **DNS privado** para resolução dentro de VNets

## NSG — Network Security Group

Firewall de rede no Azure. Define regras de entrada (inbound) e saída (outbound):
- Cada regra tem: prioridade, protocolo, porta, origem, destino, ação (Allow/Deny)
- Pode ser aplicado a uma **subnet** ou a uma **NIC** (placa de rede da VM)
- Prioridade: menor número = maior prioridade (100 é avaliado antes de 200)
