
## O que é?
Conexão de rede **privada e dedicada** entre a sua infraestrutura on-premises (empresa, datacenter) e o Azure. O tráfego **não passa pela internet pública**.

## Como funciona
Empresa/Datacenter → Parceiro ExpressRoute (operadora) → Backbone Microsoft → Azure

A Microsoft mantém parcerias com operadoras de telecomunicações que fornecem o circuito físico dedicado.

## Diferenças: ExpressRoute vs VPN Gateway

| | VPN Gateway | ExpressRoute |
|---|---|---|
| Conexão | Via internet (criptografada) | Privada (sem internet) |
| Velocidade | Até ~10 Gbps | Até 100 Gbps |
| Latência | Variável (internet) | Previsível e baixa |
| SLA | Sim | Sim (mais robusto) |
| Custo | Menor | Maior |
| Caso de uso | Acesso remoto, filiais | Migração, dados sensíveis, alta performance |

## Modelos de conectividade
- **CloudExchange co-location:** empresa no mesmo datacenter do parceiro ExpressRoute
- **Point-to-point Ethernet:** conexão direta empresa ↔ Microsoft via parceiro
- **Any-to-any (IPVPN):** integração com WAN corporativa existente

## Benefícios
- **Privacidade:** tráfego não exposto à internet
- **Velocidade:** largura de banda garantida (50 Mbps a 100 Gbps)
- **Confiabilidade:** SLA de disponibilidade
- **Compliance:** atende regulamentações que exigem conexão privada (saúde, finanças, governo)
- **ExpressRoute Global Reach:** conecta dois sites on-premises via backbone Microsoft

## O que o ExpressRoute acessa
- Todos os serviços do Azure (IaaS, PaaS)
- Microsoft 365 (com circuito específico)
- Dynamics 365

## Ponto-chave para o exame
> ExpressRoute = conexão privada dedicada com a Microsoft, **sem internet**. É mais caro, mais rápido e mais confiável que VPN. Ideal para grandes volumes de dados, compliance ou missão crítica.
