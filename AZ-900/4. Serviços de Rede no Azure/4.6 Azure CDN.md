
## O que é?
Rede distribuída de servidores (chamados de **PoPs — Points of Presence**) espalhados pelo mundo que armazenam em **cache** conteúdo estático e o entregam ao usuário a partir do servidor mais próximo geograficamente.

## Como funciona
Usuário → PoP mais próximo (cache) → se não tiver → Servidor de origem → PoP salva → Usuário recebe

Na próxima requisição do mesmo conteúdo: `Usuário → PoP (cache hit) → resposta imediata`

## Por que usar CDN?
- **Reduz latência:** conteúdo servido do servidor mais próximo do usuário
- **Reduz carga no servidor de origem:** a maioria das requisições é atendida pelo cache
- **Melhora performance global:** mesmo usuários distantes têm boa experiência
- **Escalabilidade:** absorve picos de tráfego nos PoPs

## O que pode ser cacheado
- Imagens, vídeos, arquivos de áudio
- HTML, CSS, JavaScript estáticos
- Documentos PDF, downloads
- Qualquer conteúdo que não muda frequentemente

## TTL — Time to Live
Define por quanto tempo o conteúdo fica no cache antes de ser buscado novamente na origem. Configurado pelo servidor de origem via cabeçalho `Cache-Control`.

## Azure CDN + outros serviços
- **Blob Storage:** distribuir arquivos estáticos globalmente
- **App Service / Web Apps:** cachear recursos da aplicação web
- **Azure Media Services:** streaming de vídeo

## CDN vs outros serviços de entrega
| | CDN | Traffic Manager | Front Door |
|---|---|---|---|
| Foco | Cache de conteúdo | Roteamento DNS global | CDN + LB + WAF |
| Camada | 7 (HTTP) | DNS | 7 (HTTP) |
| Cache | Sim | Não | Sim |

## Ponto-chave para o exame
> CDN serve conteúdo estático a partir de servidores geograficamente próximos ao usuário, **reduzindo latência e carga no servidor de origem**. É a solução para performance global de conteúdo.
