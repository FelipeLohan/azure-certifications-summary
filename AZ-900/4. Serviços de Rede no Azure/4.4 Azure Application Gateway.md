
## O que é?
Balanceador de carga da **Camada 7 (aplicação)** que opera no protocolo HTTP/HTTPS. Entende o conteúdo das requisições e roteia com base em URLs, cabeçalhos e outros atributos HTTP.

## Diferença fundamental vs Load Balancer

| | Load Balancer | Application Gateway |
|---|---|---|
| Camada OSI | 4 (Transporte) | 7 (Aplicação) |
| Protocolo | TCP/UDP | HTTP/HTTPS |
| Roteamento | IP + Porta | URL, path, cabeçalho, cookie |
| SSL | Não termina | Termina e re-criptografa |
| WAF | Não | Sim |

## Recursos principais

### Roteamento baseado em URL (Path-based routing)
- `/api/*` → Pool de servidores de API
- `/imagens/*` → Pool de servidores de mídia
- `/admin/*` → Pool de servidores administrativos

### Roteamento por hostname
- `app.meusite.com` → Pool A
- `api.meusite.com` → Pool B

### SSL Termination
- O Application Gateway descriptografa o HTTPS
- Processa a requisição em HTTP internamente
- Pode re-criptografar antes de enviar ao backend (end-to-end SSL)

### WAF — Web Application Firewall
- Protege contra ataques web comuns: **SQL Injection, XSS, OWASP Top 10**
- Disponível no SKU **WAF** e **WAF_v2**
- Baseado nas regras do **OWASP (Open Web Application Security Project)**

### Session affinity (Cookie-based)
- Garante que o mesmo usuário seja sempre enviado à mesma VM
- Usa cookie inserido pelo Application Gateway

## SKUs
| SKU | Características |
|---|---|
| **Standard / Standard_v2** | Roteamento HTTP/HTTPS, autoscaling (v2) |
| **WAF / WAF_v2** | Inclui firewall de aplicação web |

## Ponto-chave para o exame
> Application Gateway = Load Balancer inteligente para HTTP/HTTPS, com **WAF** incluído. É a escolha quando você precisa rotear por URL ou proteger contra ataques web.
